Java访问级别修饰符

访问级别修饰符决定了其他类是否可以使用一个字段或调用一个方法，在2个地方添加访问级别修饰符：

» 在顶层，也就是对类的访问控制，修饰符可以是public或者默认不写（包层级）

» 在成员层，对字段、方法的访问控制，可以是public、protected、private、默认不写（同包可见）

一、控制类的访问

1. public

这个类对任何其他类都是可见的，大家都可以使用。在一个源文件中，只能有一个public类，且源文件名（不包括扩展名）必和public类一致，否则编译错误。

2. 不写修饰符

不写修饰符就是采用默认控制权限，该类只对同一个包的其他类可见，在家族同辈之间共享。

二、控制成员的访问

1. public

与类相同。

2. 不写修饰符

与类相同。

3. protected

受保护的成员，同一个包可见，子类可见。（这种说法其实不清晰）

4. private

只能在该类中访问，是私有财产。

用表格来描述成员访问修饰符：

修饰符 类本身 同一个包 子类 公开
public Yes Yes Yes Yes
protected Yes Yes Yes
默认 Yes Yes
private Yes

访问级别的作用

两方面的作用：

» 别人写的类，我可以使用哪些成员

» 我写的类，别人可以使用哪些成员
